\section{Overview}

This report covers my from scratch PyTorch implementation of Faster R-CNN and a Mask R-CNN extension for the bonus. The backbone is pretrained VGG16; everything else (anchor generation, RPN, RoI pooling, detection head, training loop) is implemented from scratch. Both models were trained on Pascal VOC 2007 on a Colab T4 GPU.

Faster R-CNN achieves \textbf{59.34\% mAP}. Mask R-CNN achieves \textbf{59.62\% mAP} on bounding-box detection and \textbf{55.02\% mask mIoU} on the VOC 2007 segmentation test split (210 images), and additionally produces per-instance binary masks.

\section{Implementation}

\subsection{Backbone}

Pretrained VGG16, all conv layers up to (but not including) the final max pool (\texttt{vgg16.features[:-1]}), giving a $1/16$-resolution feature map at 512 channels. Layers 0--9 (first two conv blocks) are frozen during training.

\subsection{Region Proposal Network (Part 2)}

\paragraph{Anchor generation.}
9 anchors per feature map location: scales $\{128, 256, 512\}$ $\times$ aspect ratios $\{0.5, 1.0, 2.0\}$. A base anchor at scale $s$ and ratio $r$ has $w = s/\sqrt{r}$, $h = s\sqrt{r}$, centered on the grid cell. These are shifted across all locations using stride $= \lfloor H_\text{img} / H_\text{feat} \rfloor$. For a $600\times800$ input the feature map is roughly $38\times50$, giving about 17,100 anchors.

\paragraph{RPN head.}
A shared $3\times3$ conv (512 channels), then a $1\times1$ classification conv outputting one objectness logit per anchor ($K$ logits per location total) and a $1\times1$ regression conv outputting $4K$ box deltas. Each anchor is treated as a binary foreground/background problem and trained with \texttt{BCEWithLogitsLoss} (not $2K$ softmax scores). Weights initialized from $\mathcal{N}(0, 0.01)$, biases to zero.

\paragraph{Anchor assignment.}
Positive if IoU $\geq 0.7$ with any GT box, negative if IoU $< 0.3$, ignored otherwise. The anchor with the highest IoU per GT box is forced positive regardless of threshold, to guarantee coverage. 256 anchors sampled at 1:1 positive/negative ratio.

\paragraph{Loss.}
\begin{equation}
    \mathcal{L}_\text{RPN} = \mathcal{L}_\text{cls}^\text{RPN} + \mathcal{L}_\text{reg}^\text{RPN}
\end{equation}
$\mathcal{L}_\text{cls}$ is binary cross-entropy over sampled anchors; $\mathcal{L}_\text{reg}$ is smooth-$\ell_1$ over positive anchors only.

\paragraph{Proposal filtering.}
During training: keep pre-NMS top 12k by objectness, apply NMS at IoU 0.7, keep post-NMS top 2k. During testing: pre-NMS top 6k, same NMS, post-NMS top 300. Boxes are clamped to the image boundary before degenerate-box removal because clamping itself can create zero-width boxes.

\subsection{RoI Feature Extraction (Part 3)}

RoI Pooling extracts $7\times7$ feature maps per proposal. The spatial scale is $\approx 1/16$ for VGG16 (no final max-pool), computed dynamically as $\min(H_\text{feat}/H_\text{img},\, W_\text{feat}/W_\text{img})$ to handle rounding after image resizing. GT boxes are appended to proposals during training. Each proposal is matched to its highest-IoU GT box; IoU $\geq 0.5$ is positive, $< 0.5$ is background. 128 proposals sampled at 1:4 positive/negative ratio.

\subsection{Detection Head (Part 4)}

Flatten $\to$ FC(25088, 1024) $\to$ ReLU $\to$ FC(1024, 1024) $\to$ ReLU, then a classification head FC(1024, 21) and regression head FC(1024, $21\times4$). Classification weights from $\mathcal{N}(0, 0.01)$, regression from $\mathcal{N}(0, 0.001)$ (tighter, following the paper). At inference: pick the top non-background class, decode its box, then per-class NMS at 0.3 with a 0.05 score threshold.

\subsection{Full Model and Training (Part 5)}

Images are normalized with ImageNet mean/std and resized so the shorter side is 600 px (max 1000), preserving aspect ratio. GT boxes are scaled accordingly. At inference, predicted boxes are mapped back to original image coordinates.

Total loss:
\begin{equation}
    \mathcal{L} = \mathcal{L}_\text{cls}^\text{RPN} + \mathcal{L}_\text{reg}^\text{RPN} + \mathcal{L}_\text{cls}^\text{FRCNN} + \mathcal{L}_\text{reg}^\text{FRCNN}
\end{equation}

SGD with momentum 0.9, weight decay $5\times10^{-4}$, LR $10^{-3}$ decayed $\times0.1$ at epochs 14 and 18, for 20 epochs.

\section{Bonus: Mask R-CNN Extension}

\subsection{Architecture Changes}

Mask R-CNN adds a mask branch in parallel with the detection head. The detection branch is unchanged; the mask branch only fires for foreground proposals and only predicts the mask for the GT class.

\paragraph{RoI Align.}
The mask branch uses RoI Align at $14\times14$ resolution. RoI Align avoids the double quantization in RoI Pooling by using bilinear interpolation at sub-pixel sampling points:
\begin{equation}
    f(x, y) = \sum_{i,j} w_{ij} \cdot f(x_i, y_j)
\end{equation}
This eliminates the misalignment that makes pixel-level prediction harder with standard pooling.

\paragraph{Mask head.}
A small FCN on top of the $14\times14$ RoI Align output:
\begin{equation}
    \underbrace{14 \times 14}_{\text{RoI Align}} \xrightarrow{\times 4\ \text{conv}(256,\, 3\times3)} \xrightarrow{\text{deconv}(2\times)} \underbrace{28 \times 28}_{\text{mask logits}}
\end{equation}
4 $\times$ [Conv2d(in, 256, $3\times3$) + ReLU], ConvTranspose2d(256, 256, $2\times2$, stride=2) + ReLU, Conv2d(256, $C$, $1\times1$). Outputs $C$ binary mask logits at $28\times28$ per RoI.

\paragraph{Mask targets.}
For each foreground proposal, the GT binary mask is cropped to the matched GT bounding box and resized to $28\times28$ using RoI Align at spatial scale 1.0. This gives a target aligned with the predicted mask.

\subsection{Loss}

\begin{equation}
    \mathcal{L} = \mathcal{L}_\text{cls}^\text{RPN} + \mathcal{L}_\text{reg}^\text{RPN} + \mathcal{L}_\text{cls} + \mathcal{L}_\text{reg} + \mathcal{L}_\text{mask}
\end{equation}

Binary cross-entropy over foreground proposals only, using only the mask channel for the GT class:
\begin{equation}
    \mathcal{L}_\text{mask} = -\frac{1}{N_\text{fg}} \sum_{i \in \text{fg}} \text{BCE}\!\left(\hat{m}_i^{c_i},\, m_i^*\right)
\end{equation}
Only supervising the GT class channel means wrong-class masks never get a gradient, so the head learns class-specific shapes.

\subsection{Dataset}

VOC 2007 has instance segmentation masks for 632 images in \texttt{SegmentationObject/}. The PNG encodes instance indices: pixel value $k > 0$ is the $k$-th object in the XML annotation (1-indexed), 255 is void/boundary. All 5,011 training images are used; the mask loss is skipped for images without segmentation annotations.

\subsection{Training}

\section{Challenges}

\paragraph{Stride calculation.}
The stride has to be computed from the actual image/feature-map size ratio each forward pass, not hardcoded to 16, because resized images aren't always exact multiples. Anchor centers also need to be offset by stride/2 to sit in the middle of the grid cell.

\paragraph{Proposal filtering order.}
I ran into degenerate-box issues until I realized clamping proposals to the image boundary can itself create zero-width boxes. Degenerate removal has to happen after clamping.

\paragraph{Class-specific regression.}
The detection head predicts $21\times4$ deltas. At training only the GT class deltas are supervised; at inference only the top-scoring class deltas are decoded. Getting the index slicing right (\texttt{box\_preds[range(N), labels]}) took more debugging than I expected.

\paragraph{Mask target generation.}
To get $28\times28$ GT masks aligned with each proposal, I crop the full-resolution GT mask to the GT bounding box and resize using RoI Align at spatial scale 1.0. Plain bilinear resize doesn't respect the box boundary the same way.

\paragraph{VOC segmentation matching.}
The integer instance indices in the \texttt{SegmentationObject} PNGs correspond to annotation order in the XML (1-indexed). A few images have edge cases where this mapping breaks; I added a bounding-box-overlap fallback for those.

\paragraph{CUDA vs MPS.}
Final training ran on Colab CUDA. The code also supports Apple MPS as a fallback for local development, though I hit one or two ops during testing that weren't fully MPS-compatible in my PyTorch version and had to verify everything worked before switching to Colab.

\section{Conclusion}

Both models were implemented from scratch and trained on VOC 2007. Faster R-CNN hits 59.34\% mAP, which is reasonable for a VGG16 single-scale model. Mask R-CNN matches that at 59.62\% mAP and adds instance segmentation at 55.02\% mask mIoU. Closing the gap to the $\sim$70\% torchvision reference would mostly come from switching to a ResNet-50 + FPN backbone, not from changes to the training procedure.
